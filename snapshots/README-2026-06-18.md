## 🧭 The Landscape in One Screen

The clearest mental model: **the CLI coding agents are the substrate; the OpenClaw-style always-on personal assistants are the new application layer that orchestrates them.** As of June 18, 2026, the landscape has consolidated around four overlapping layers, with the headline story being a genuinely new product category — self-hosted, chat-reachable, persistent, proactive, computer-controlling assistants.

| # | Layer | Key fact | Headline tools |
|---|-------|----------|----------------|
| 1 | **CLI coding agents** | Table-stakes | Claude Code, Codex CLI, Gemini CLI, Cursor, Copilot, Kiro · OSS: Aider, Crush, Qwen Code, OpenCode (≈176k★) |
| 2 | **OpenClaw-style always-on assistants** ⭐ | *The new category* | OpenClaw (≈370k★), Memoh, OwnPilot, Lethe · Proprietary: Claude Cowork+Dispatch, Perplexity Computer, Gemini Spark |
| 3 | **Computer-use / browser agents** | Crossed human parity *(self-reported)* | Anthropic CU, ChatGPT agent, Simular Agent S3, Browser Use (≈99k★), OpenHands, OmniParser v2 |
| 4 | **Agent frameworks / SDKs** | Three-vendor race | OpenAI Agents SDK, Claude Agent SDK + Skills (≈152k★), Google ADK 2.0 · LangGraph, CrewAI, Cline |

**Three macro-trends cut across all four layers:**

1. **Absorption** — standalone browser-agent products are being folded into first-party assistants (Operator → ChatGPT agent; Project Mariner → Gemini Spark).
2. **"Skills" becoming the standard capability interface** — Anthropic Agent Skills, OpenClaw `SKILL.md`, and Cline/Cursor/Google formats are converging on the same capability-pack shape.
3. **A self-hosted-vs-managed split** — OSS offers data sovereignty + self-hackability at the cost of setup and real security risk; proprietary offers polish + a kill switch but your data lives on vendor infra.

## ⌨️ CLI Coding Tools

The most mature and crowded segment. Splits into **vendor CLIs** (closed, subscription-tied), **open-source multi-provider agents**, and **AI-native terminals**. **Pricing has converged on subscription-plus-credits**; the free-tier winner is **Gemini CLI**.

### Vendor CLIs (proprietary)

- **[Claude Code](https://github.com/anthropics/claude-code)** — Reasoning-quality leader. Fully agentic: multi-file edits, sandboxed bash, subagents, MCP, hooks, headless + background agents. Split-credit pricing since Jun 15, 2026. `Proprietary` · Anthropic-only models
- **[Codex CLI](https://github.com/openai/codex)** — Sandboxed agentic execution; Rust rewrite. Bundled with ChatGPT plans or API-metered. Top vendor CLI on Terminal-Bench. `Apache-2.0` · Rust · ≈91.7k★ *≈unverified*
- **[Gemini CLI](https://github.com/google-gemini/gemini-cli)** — **Best free tier** of any vendor CLI (60 req/min, 1,000/day). Subagents + Plan mode, 1M-token context. `Apache-2.0` · TS · ≈105k★ *≈unverified*
- **[Cursor CLI](https://cursor.com/docs/cli/overview)** — Agent / Plan / Ask modes + Cloud Agent handoff (`&`) + headless (`-p`). Tightest parity with the Cursor editor. `Proprietary` · Teams pricing refreshed Jun 2026
- **[GitHub Copilot CLI](https://github.com/features/copilot/cli)** — Deepest GitHub-native integration. `/fleet` parallel subagents, autopilot, `AGENTS.md` skills, native MCP. Usage-metered AI Credits since Jun 1, 2026. `Proprietary` · multi-model via `/model`
- **[Kiro CLI](https://kiro.dev/docs/cli/migrating-from-q/)** — Replaces the **deprecated OSS Amazon Q Developer CLI** (OSS → closed). NL→bash, >500-tool autocomplete, custom agents. First-party AWS/Bedrock/IAM. `Proprietary` · Bedrock-hosted

### Open-source multi-provider agents

- **[Aider](https://github.com/Aider-AI/aider)** — Maturest community tool. Git-integrated pair-programming loop, repo map, auto-commits, lint/test after edits, watch mode. Hosts the widely-cited LLM coding leaderboard. `Apache-2.0` · Python · ≈46.4k★ *≈unverified*
- **[Crush](https://github.com/charmbracelet/crush)** — Best-in-class TUI/UX, broadest provider matrix out of the box (Anthropic, OpenAI, Gemini, Groq, OpenRouter, Bedrock, Vertex, Azure, Ollama). `OSS` · Go · *(lineage "rebranded from OpenCode" unverified)*
- **[Qwen Code](https://github.com/QwenLM/qwen-code)** — Self-described Claude-Code feature-parity: subagents, agent teams, auto-memory/skills/hooks, MCP, plan mode, computer use. Pairs an open model (Qwen3-Coder) with an open framework. `Apache-2.0` · Node.js · ≈25.3k★ *≈unverified*
- **[OpenCode](https://github.com/anomalyco/opencode)** — **Highest-traction OSS terminal coding agent.** 75+ providers, multi-session parallel agents, desktop beta. *(Two "OpenCode" lineages — see [caveats](#-methodology--caveats).)* `OSS` · TS · **≈176k★** *≈unverified*

### AI-native terminals & legacy

- **[Warp](https://github.com/warpdotdev/warp)** — Not a coding agent — an AI-native terminal that *replaces the shell itself* (NL→bash, command generation, block-based dev env). Client open-sourced under AGPL-3.0 (~May 2026); server/AI orchestration proprietary. ≈37k★ *≈unverified*
- **Continue** — *Archived* after a final 2.0.0 release (VS Code ext, `cn` CLI, JetBrains plugin). See [Shutdowns](#-shutdowns--archived-dont-recommend-these).
- **Sourcegraph Cody CLI** — Effectively legacy; modern successor is **Amp** (`@ampcode/cli`).

> **CLI takeaways:** pricing is converging on subscription-plus-credits. The OSS field has standardized on multi-provider + BYO-key. Discontinuations matter (Q CLI → Kiro OSS→closed; Continue archived; Cody → Amp) — don't recommend the deprecated ones.

## 🤖 Always-On Personal AI Assistants — *the focal category*

> The defining new category of 2026: **open-source, self-hosted, always-on personal AI assistants** — persistent agents that live on your own hardware, are reachable from the chat apps you already use, remember you across days, proactively initiate contact, control your computer, and can extend themselves with "skills."

A tool qualifies for this category only if it satisfies all seven criteria: (a) self-hosted on user-owned hardware · (b) messaging-first reachability from existing chat apps · (c) persistent cross-session memory · (d) proactive heartbeat/cron behavior · (e) extensible skills with a registry · (f) broad multi-provider + local model routing · (g) computer/tool control.

### The OpenClaw wave (self-hosted OSS)

| Project | Self-host? | Chat | Proactive? | Computer control | Skills | License · Lang | Stars |
|---|---|---|---|---|---|---|---|
| **[OpenClaw](https://github.com/openclaw/openclaw)** | ✅ Mac/Linux/Win/RPi/Docker/K8s | ✅ 20+ | ✅ 30-min heartbeat + cron | ✅ browser + macOS GUI | ✅ `SKILL.md` + TS plugins + ClawHub | `MIT` · TS+Swift | **≈370k+** *≈approx/self-reported* |
| **[Memoh](https://github.com/memohai/Memoh)** | ✅ container-per-agent | ✅ TG, Discord, Lark, WeChat, QQ, Email, Web | ✅ scheduled + heartbeat | ✅ Browser Use + container desktop | ✅ Skills + Supermarket + MCP | `AGPL-3.0` · Go+Vue+Rust | *(unverified)* |
| **[OwnPilot](https://github.com/ownpilot/ownpilot)** | ✅ Docker/ghcr | ✅ Telegram + WhatsApp + web | ✅ "Heartbeat Lifecycle", soul agents, crews | ✅ Browser Agent + edge/IoT | ✅ 250+ tools, MCP client+server, `SKILL.md` | `MIT` · TS | *(unverified; v0.7.2)* |
| **[Lethe](https://github.com/atemerev/lethe)** | ✅ rootless container / `--yolo` native | ⚠️ Telegram + HTTP/SSE only | ✅ heartbeat + "default-mode network" | ✅ browser, shell/PTY, FS | ✅ tool registry (described as self-modifying) | `MIT` · Rust (~50 MB) | *(unverified)* |

- **OpenClaw** (`openclaw/openclaw`, by **Peter Steinberger / @steipete**) is the reference implementation of the category. A self-hosted "Gateway" daemon bridges 20+ chat surfaces (WhatsApp, Telegram, Slack, Discord, iMessage, …) to an LLM, with local markdown memory (git-backed), a 30-minute proactive "heartbeat," a self-hackable `SKILL.md` skills system hot-reloaded by a file watcher, computer control, and broad model routing. Built Nov 2025 as a 1-hour prototype; Steinberger joined OpenAI Feb 2026; project moving to an independent open-source foundation sponsored by OpenAI/GitHub/NVIDIA/Vercel.
- **Memoh** is the closest structural peer — each agent gets its own containerized "cloud computer." Note the **AGPL-3.0** copyleft vs OpenClaw's MIT.
- **OwnPilot** is explicitly OpenClaw-derived (`SKILL.md` format, "Claw" naming); strongest feature overlap incl. edge/IoT control.
- **Lethe** is the most architecturally distinctive — a single ~50 MB Rust binary with a brain-inspired multi-actor design (cortex/hippocampus/brainstem/default-mode-network) and a loop its README describes as self-modifying. Borderline category fit (Telegram + HTTP/SSE only).

### Proprietary equivalents

Recent tech press frames these as the industry's answer to OpenClaw. *(Note: "response" here is press inference — Ars Technica calls Perplexity Computer "a more secure OpenClaw" — not a confirmed vendor statement. Perplexity Computer launched Feb 2026 contemporaneously with OpenClaw's virality, so direct causation is not established.)*

- **[Claude Cowork + Dispatch](https://claude.com/docs/cowork/overview)** — GA Apr 9, 2026. Full desktop control (macOS GA; Windows "soon") + browser. Dispatch = one persistent conversation messageable from your phone that runs while you're away. Ships finished deliverables (decks, spreadsheets, reports). `Proprietary`
- **[Perplexity Computer / Personal Computer](https://www.perplexity.ai/hub/blog/everything-is-computer)** — Model-agnostic orchestrator of ~20 models. Personal Computer GA on Mac May 7, 2026 on a dedicated Mac mini, mobile remote control, kill switch, audit trail. `Proprietary`
- **[Gemini Spark](https://blog.google/innovation-and-ai/products/gemini-app/next-evolution-gemini-app/)** — "24/7 personal AI agent" with macOS app integration; announced I/O 2026. Successor to shut-down Project Mariner. `Proprietary` · `announced`
- **Manus** (Meta) — Cross-platform chat-reachable autonomous agent; "My Computer" desktop mode (Mar 2026), but **Meta is moving to unwind the deal** as of Jun 2026 after a Beijing divestiture order. Future in flux. `Proprietary` · `uncertain`

### Adjacent (explicitly NOT the same category)

- **[Goose](https://github.com/block/goose)** — General-purpose developer/desktop agent, 15+ providers, 70+ MCP extensions; now under the **Linux Foundation's Agentic AI Foundation (AAIF)**. Has `schedule`/`session` commands but is **not messaging-first**. A peer in the broader OSS-agent space, not in OpenClaw's niche. `Apache-2.0` · Rust
- **[SuperAGI](https://github.com/TransformerOptimus/SuperAGI)** — 2023-era web-UI autonomous-agent framework with concurrent agents and toolkits marketplace. Web-UI-centric, not chat-reachable. `Apache-2.0` · Python+Next.js · ≈15k★ *≈approx*
- **[Daytona](https://github.com/daytonaio/daytona)** — Secure elastic infra/sandbox for running AI code; it's the *substrate* an agent like OpenClaw can run inside (Daytona's docs include a "Run OpenClaw in a secure sandbox" guide), not an assistant itself.

## 🖱️ Computer-Use & Browser Agents

Computer-use has stratified into three layers: an **API/builder layer**, a **first-party consumer-agent layer**, and a fast-moving **benchmark story**.

### Builder / API layer

- **[Anthropic Computer Use](https://platform.claude.com/docs/en/agents-and-tools/tool-use/computer-use-tool)** — Canonical "screenshot → coordinate → action" loop in a Docker Linux+X11+VNC container. De-facto builder starting point. Beta (header `computer-use-2025-11-24`); powers Opus 4.5–4.8 / Sonnet 4.6. Production best-practices published May 13, 2026. `API proprietary / demo OSS`
- **[OpenAI CUA](https://openai.com/index/computer-using-agent/)** — The Computer-Using Agent model, exposed via the Agents SDK / Responses API `computer` tool. Originally powered Operator. `Proprietary`
- **[Microsoft OmniParser v2](https://www.microsoft.com/en-us/research/articles/omniparser-v2-turning-any-llm-into-a-computer-use-agent/)** — **Not an agent** — a pure-vision GUI-grounding module that "tokenizes" screenshots to turn *any* LLM into a GUI agent. Paired with OmniTool (dockerized Win11 VM). `OSS` · Python · ≈25k★ *≈unverified*
- **[Simular Agent S3](https://www.simular.ai/articles/agent-s3)** — First **vendor-reported** OSWorld crossing: 72.60% (best-of-N; single-pass 66%). Full desktop (mac/Linux/Win) + Android. Raised $21.5M (Felicis, Nvidia NVentures). `Apache-2.0` · Python · ≈11.9k★ *≈unverified* *(see benchmark caveat below)*
- **[Browser Use](https://github.com/browser-use/browser-use)** — Widely-used OSS browser-agent library (Playwright). CLI 2.0 uses direct Chrome DevTools Protocol (~50 ms latency); **0.13.0 (Jun 8 2026) introduced a Rust-backed beta**. Commercial **Browser Use Box** is explicitly "your 24/7 Claude agent from Telegram/SSH" (direct OpenClaw-pattern overlap). `MIT` · Python · ≈99.3k★ *≈unverified*
- **[Skyvern](https://github.com/Skyvern-AI/skyvern)** — Enterprise browser-RPA replacement. Planner-Agent-Validator architecture, native CAPTCHA + 2FA, SOC2/HIPAA. Workflow engine, not chat-reachable. `AGPL-3.0` · Python · ≈21.9k★ *≈unverified*
- **[OpenHands](https://github.com/OpenHands/OpenHands)** — Self-hosted "developer control center" running its own agent *or* any ACP-compatible agent across local/Docker/VM/cloud. Ships SDK + CLI + GUI + **Agent Canvas** (scheduled tasks, Slack/GitHub/Linear automations). Formerly OpenDevin. `MIT core` · Python · ≈77.5k★ *≈unverified*

### First-party consumer agents

- **[Claude Cowork + Dispatch + Claude in Chrome](https://claude.com/docs/cowork/overview)** — GA Apr 9, 2026 on all paid plans. Full desktop control + browser. Dispatch = persistent session, messageable from phone, runs in background. *(covered under [Always-On](#proprietary-equivalents))* `Proprietary` · `ga`
- **[ChatGPT agent](https://openai.com/index/introducing-operator/)** (ex-Operator/CUA) — Operator was **shut down Aug 31, 2025** and folded into ChatGPT "agent mode." **Workspace Agents** (Codex-powered, cloud, Slack-reachable, scheduled/cloud persistence) launched Apr 22, 2026. `Proprietary`
- **[Gemini Agent / auto-browse / Gemini Spark](https://www.aichatdaily.com/ai-news/google-shuts-down-project-mariner)** — **Project Mariner was shut down May 4, 2026** (browser automation proved too brittle: CAPTCHAs, anti-bot, layout drift). Successors: Gemini in Chrome + auto-browse (shipping), Gemini Spark (I/O 2026), AI Mode information agents (rolling out Jun 2026). `Proprietary`
- **[Perplexity Computer / Personal Computer](https://www.perplexity.ai/hub/blog/everything-is-computer)** — 24/7 digital proxy on a dedicated Mac mini. *(covered under [Always-On](#proprietary-equivalents))* `Proprietary`
- **[Apple Siri AI](https://www.apple.com/newsroom/)** — **Deliberately NOT vision-based computer use.** Reasons over on-screen content + personal context but acts via structured **App Intents**, not pixel clicks. Always-on across iPhone/iPad/Mac/Watch, privacy-first. WWDC26. `Proprietary` · on-device · `beta`
- **Manus** (Meta) — Cross-platform chat-reachable autonomous agent. *(covered under [Always-On](#proprietary-equivalents); status uncertain)*

### ⚠️ Benchmark reality check (OSWorld)

The headline numbers are moving fast and **almost entirely vendor-self-reported**, and the aggregators disagree with each other by ~10 points:

| Source | Top entry | Top score | Notes |
|---|---|---|---|
| **[AwesomeAgents](https://awesomeagents.ai/leaderboards/computer-use-leaderboard/)** | GPT-5.4 | **75.0%** | Fable/Mythos not listed |
| **[Steel.dev](https://leaderboard.steel.dev/leaderboards/osworld/)** | Claude Opus 4.8 | **83.4%** | No entry exceeds 85%; Opus 4.6 marked *self-reported* |
| **[BenchLM](https://benchlm.ai/benchmarks/osWorldVerified)** | Fable 5 / Mythos 5 | **~85%** | Fable/Mythos reportedly government-suspended mid-Jun 2026 |
| Human baseline ([OSWorld paper](https://os-world.github.io/)) | — | **72.36%** | The original benchmark |
| **[Simular Agent S3](https://www.simular.ai/articles/simulars-computer-use-agent-outperforms-humans)** (vendor) | Agent S3 | **72.60%** | Best-of-N; **single-pass 66%**; not on any independent leaderboard |

> **Bottom line:** computer-use has reached human-baseline *territory*, but no single clean, independently-verified, currently-accessible number establishes that it has *surpassed* humans. Read all OSWorld-Verified figures as vendor self-reports that vary by ~10 points across aggregators.

## 🧩 Agent Frameworks & SDKs

### The three-vendor SDK race

- **[OpenAI Agents SDK](https://github.com/openai/openai-agents-python)** — Productionized successor to Swarm. Agents, Sandbox Agents, Handoffs, Tools (fn/MCP/hosted), Guardrails, HITL, Sessions, Tracing, Realtime voice agents. 100+ LLMs. `MIT` · Python, JS/TS · ≈27.2k★ *≈unverified*
- **[Claude Agent SDK](https://github.com/anthropics/claude-agent-sdk-python)** — Renamed Claude Code SDK; programmatic access to the same agent loop/tools/context management as Claude Code. `MIT` · Python, TS · ≈7.3k★ *≈unverified (py)*
- **[Anthropic Agent Skills](https://github.com/anthropics/skills)** — **The bigger story.** Reference implementation of the Agent Skills standard (`SKILL.md` capability packs loaded dynamically), widely cited as having **"surged past MCP"** as the de-facto capabilities layer. `Apache-2.0 (skills)` · Markdown · **≈152k★** *≈unverified*
- **[Google ADK 2.0](https://github.com/google/adk-python)** — Code-first build/eval/deploy. ADK 2.0 added a graph-based Workflow Runtime (routing, fan-out/fan-in, loops, HITL) + Task API for agent-to-agent delegation. Gemini-optimized but model-agnostic. `Apache-2.0` · Python, Go, Java, TS · ≈20.2k★ *≈unverified (py)*
- **Microsoft consolidation:** **[AutoGen](https://github.com/microsoft/autogen)** is in **maintenance mode**; official successor **[Microsoft Agent Framework](https://github.com/microsoft/agent-framework)** hit **1.0 GA**, merging AutoGen + Semantic Kernel with A2A + MCP interop. `CC-BY-4.0 → MIT` · ≈58.9k★ (AutoGen) / ≈11.4k★ (MAF) *≈unverified*

### Orchestration layer

- **[LangGraph](https://github.com/langchain-ai/langgraph)** — **1.0 GA October 2025.** Low-level stateful/durable graph orchestration with first-class HITL interrupts + memory. Higher-level **Deep Agents** harness on top. `MIT` · Python, JS/TS · ≈35.1k★ *≈unverified*
- **[CrewAI](https://github.com/CrewAIInc/CrewAI)** — Role-based **Crews** (autonomous agent teams) + deterministic **Flows**. Standalone since v1.14 (no LangChain dep). Fortune-500 traction. `MIT` · Python · ≈53.8k★ *≈unverified*
- **[PocketFlow](https://github.com/The-Pocket/PocketFlow)** — Deliberately **100-line** minimalist graph framework, zero dependencies. Positioned for "let Cursor/Claude build agents on top." `MIT` · Python, TS, Java, C++, Go · ≈11k★ *≈unverified*
- **[PydanticAI](https://github.com/pydantic/pydantic-ai)** — "FastAPI-feeling," fully type-safe, model-agnostic. Tight OTel/Logfire observability. v2.0 beta. `MIT` · Python · ≈17.8k★ *≈unverified*
- **[smolagents](https://github.com/huggingface/smolagents)** — Agents that "think in code" — CodeAgent writes Python (sandboxed via E2B/Modal/Docker) vs JSON ToolCallingAgent. ~1k-line core. `Apache-2.0` · Python · ≈27.9k★ *≈unverified*
- **[LlamaIndex](https://github.com/run-llama/llama_index)** — RAG-native "document agent" platform. Multi-agent via AgentWorkflow + custom planners. `MIT` · Python, TS · ≈50.2k★ *≈unverified*

### Editor-agent runtimes & managed platforms

- **[Cline](https://github.com/cline/cline)** — Pioneer agentic VS Code extension, repositioned as **one runtime across IDE + terminal + SDK** (CLI 3.0 May 2026 on a new SDK/TUI). 8M+ installs. `Apache-2.0` · TS · **≈63k★** *≈unverified*
- **[Roo Code](https://github.com/RooCodeInc/Roo-Code)** — **ARCHIVED 2026-05-15** (final v3.54.0). Cline fork w/ custom modes; team pivoted to "Roomote" (Slack-first cloud). Successors: Zoo Code, Kilo Code. See [Shutdowns](#-shutdowns--archived-dont-recommend-these). `Apache-2.0` · ≈24.2k★ *≈unverified*
- **[Devin / Cognition](https://cognition.ai)** — Managed "autonomous software engineer." **Devin Desktop** (Jun 2026) on a Rust "Devin Local" engine supporting ACP + third-party agents. "Managed Devins" = a coordinator spawning parallel child agents each in its own VM. `Proprietary`

## 🔬 Methodology & Caveats

Read this before citing anything here. This is a **landscape map, not a benchmark study.**

- **Star counts are popularity, not quality** — and most are marked `*≈unverified*` (pulled from tracker/repo pages, not re-pulled from the GitHub API on 2026-06-18). They drift within days. The OpenClaw figure (~370k+) is **self-reported and approximate**; re-pull from `api.github.com/repos/openclaw/openclaw` before any hard citation.
- **The OSWorld "human parity" story is not clean.** Vendor self-reports vary by ~10 points across aggregators (75% / 83.4% / 85%); the only claimed crossing (Agent S3, 72.60%) is vendor-reported, best-of-N (single-pass 66%), and on no independent leaderboard; the cited top models (Fable 5 / Mythos 5) are reportedly government-suspended as of mid-June 2026.
- **The OpenClaw "self-hackable / AGI-like" rhetoric is ahead of the verified evidence.** The skill-creator + hot-reload machinery is real and code-confirmed; the reliable in-practice quality of agent-authored skills is **not independently benchmarked.** Treat enthusiastic testimonials as marketing.
- **Security incidents are real, but specific figures are unconfirmed.** The frequently-cited severity figures ("Cisco found 2 critical of 9 findings of silent data exfiltration; Wiz disclosed a misconfigured DB exposing ~1.5M API keys / ~35k emails") appear only in a secondary profile and are not confirmed against a primary Cisco/Wiz disclosure.
- **Vendor causation is not confirmed** — proprietary launches framed as "responses to OpenClaw" are press inference, not vendor statements.
- **Two "OpenCode" lineages need disambiguation:** the major `sst/opencode` → `anomalyco/opencode` (~176k★) vs the project Charm's Crush was claimed to be "rebranded from" (the current Crush README does not contain that phrase — unverified).
- **Manus's status is genuinely uncertain** (product reportedly live while Meta unwinds the acquisition).
